About the Project

LANCast is an SDN-driven one-to-many screen broadcasting system built for offline LAN environments. A Java-based broadcaster application sends multicast video frames to many viewer clients, and an SDN controller (Ryu + OpenFlow) intelligently manages the network traffic to ensure efficient delivery, low latency, and optimized bandwidth usage.

Key Features

Offline Streaming

Broadcast your screen to multiple devices without needing internet.

SDN Traffic Optimization

Uses SDN controller and OpenFlow to manage network traffic efficiently.

Multicast Delivery

One hosted stream is replicated to many viewers using UDP multicast.

System Architecture

Broadcaster (Java App)
        |
        | UDP Multicast
        v
OpenFlow Switch  <---->  SDN Controller (Ryu)
        |
        v
Multiple Receivers (Java App)

The SDN Controller monitors and manages packet flows between broadcaster and receivers using OpenFlow.

How It Works

  • Java broadcaster captures screen frames and encodes them.
  • Frames are sent via UDP multicast to a multicast group.
  • Receivers subscribe to the multicast group to display the broadcast.
  • SDN Ryu Controller detects traffic and installs OpenFlow rules.
  • Switches forward packets based on controller instructions for optimal performance.
